# Cal Poly Conference and Event Planning

Frontend application for the Cal Poly Conference and Event Planning website, built with Next.js, FaustWP, and WordPress as a headless CMS.

This repository powers a content-driven marketing site with:

- A WordPress-driven homepage and standard pages
- Blog-style posts and project/case-study content
- Site search
- Dynamic SEO metadata, `robots.txt`, and `sitemap.xml`
- Custom footer and menu locations for the Cal Poly Conferences information architecture

The canonical production site is `https://www.calpolyconferences.org`.

## Stack

- Next.js 14
- React 18
- FaustWP
- WPGraphQL / headless WordPress
- Apollo Client
- Sass modules

## Requirements

- Node.js 18+ recommended
- npm
- A WordPress instance with FaustWP and WPGraphQL enabled

The `package.json` engine range is permissive, but this app is on Next.js 14, so current Node LTS is the safer baseline for local development and deployment.

## Environment Variables

Create `.env.local` from `.env.local.sample`.

```bash
cp .env.local.sample .env.local
```

Required variables:

```env
NEXT_PUBLIC_WORDPRESS_URL=https://your-wordpress-site.example
```

Optional variables:

```env
FAUST_SECRET_KEY=your_faust_secret
```

Notes:

- `NEXT_PUBLIC_WORDPRESS_URL` is used by FaustWP and by the custom sitemap route.
- `FAUST_SECRET_KEY` is required for Faust preview/auth flows when your local environment needs it.

## WordPress Requirements

This frontend expects a connected WordPress backend with:

- FaustWP
- WPGraphQL
- Content for pages, posts, projects, and testimonials
- Menu assignments matching the locations used in this app

Configured menu locations in this repo:

- `PRIMARY`
- `FOOTER`
- `FOOTER_SECONDARY`
- `FOOTER_TERTIARY`
- `RESOURCES_FOOTER`
- `QUICK_FOOTER`
- `ABOUT_FOOTER`

If these menu locations are missing or not assigned in WordPress, navigation and footer content will render incorrectly or fall back inconsistently.

## Site Behavior

Important application behavior implemented in this repo:

- The homepage is rendered through the WordPress `front-page` template.
- Standard WordPress pages and posts render through Faust templates.
- `/projects` exposes project-style content backed by a custom project template/plugin flow.
- `/search` is client-driven, query-string based, and marked `noindex`.
- `/robots.txt` and `/sitemap.xml` are generated by Next.js routes.
- SEO metadata is assembled from page content plus config in [`app.config.js`](/Users/bchenowe/Sites/conferenceeventplanning/app.config.js).

## Deployment Notes

- The app is configured with `withFaust(...)` in [`next.config.js`](/Users/bchenowe/Sites/conferenceeventplanning/next.config.js).
- Remote image allowlists are explicitly defined there and must stay in sync with the active WordPress/media domains.
- The canonical site URL and organization schema values live in [`app.config.js`](/Users/bchenowe/Sites/conferenceeventplanning/app.config.js).

If the production domain, WordPress origin, or media host changes, update those files before deploying.
